Kotlin Multiplatform / JVM / Android

Layered JSON config. Typed API in shared code.

Kayan merges base config with optional overrides and generates Kotlin values. Shared code reads typed properties directly — no platform-specific wiring.

  2. Custom config default value

Quick start

Four steps to typed config

1 Add your JSON config

{
  "flavors": {
    "prod": {
      "api_base_url": "https://api.example.com",
      "feature_search_enabled": false
    },
    "dev": {
      "api_base_url": "https://dev.example.com",
      "feature_search_enabled": true
    }
  },
  "brand_name": "Example App"
}

2 Apply the plugin & declare schema

plugins {
    // Use any supported Kotlin plugin:
    // kotlin("multiplatform"), kotlin("jvm"), or kotlin("android")
    kotlin("multiplatform") version "<kotlin-version>"
    id("io.github.mohamadjaara.kayan") version "<kayan-version>"
}

kayan {
    packageName.set("sample.generated")
    flavor.set("prod")

    schema {
        string("api_base_url", "API_BASE_URL",
               required = true)
        boolean("feature_search_enabled",
                "FEATURE_SEARCH_ENABLED")
        string("brand_name", "BRAND_NAME")
    }
}

Add mavenCentral() to pluginManagement.repositories when resolving the published plugin.

3 Build to generate

./gradlew generateKayanConfig

Generated source lands in build/generated/kayan/kotlin and is wired into the appropriate source set automatically (commonMain for KMP, main for JVM/Android).

4 Use generated values

import sample.generated.SampleConfig

val baseUrl = SampleConfig.API_BASE_URL
val search  = SampleConfig.FEATURE_SEARCH_ENABLED
val brand   = SampleConfig.BRAND_NAME

What you get

Built for config that changes by flavor, environment, or brand

Use cases

  • Gradual migration away from BuildConfig
  • White-label products with one shared base config
  • Compile-time feature flags and endpoints
  • Generated JSON Schema and Markdown docs

Schema kinds

  • string, boolean, int, long, double
  • stringMap, stringList, stringListMap
  • enumValue for normalized enum generation
  • custom adapters for consumer-owned types

White-labeling

Build branded apps from one repo

Base app repo

app/
  default.json
  build.gradle.kts

Shared code, shared schema, one default config.

Brand source outside the repo

branding/
  wafflewizard.json
  bananabeacon.json

Another checkout, object storage, or a raw URL can own the brand overrides.

Compile-time fetch

curl -o /tmp/wafflewizard.json https://example.com/mobile-branding/wafflewizard.json
./gradlew generateKayanConfig \
  -PbrandConfigPath=/tmp/wafflewizard.json \
  -PkayanFlavor=prod

Another repo also works

./gradlew generateKayanConfig \
  -PbrandConfigPath=../branding/bananabeacon.json \
  -PkayanFlavor=staging
